### 🔧 技术实施

#### 1. 注意力层性能基准测试设计

##### 1.1 测试架构
- **文件位置**: `benchmarks/attention_performance.c`
- **构建集成**: 更新 `tests/CMakeLists.txt`，添加 `test_attention_performance` 目标
- **计时机制**: 跨平台高精度计时器（Windows: `QueryPerformanceCounter`，其他: `gettimeofday`）
- **配置系统**: 支持多组参数配置的批量测试

##### 1.2 测试配置
设计了4种典型配置，覆盖不同应用场景：
1. **小型配置** (2x4x32x4): 适用于单元测试和调试
   - Batch size: 2, Sequence length: 4, Hidden size: 32, Heads: 4
   - 迭代次数: 100次，确保统计显著性

2. **中型配置** (4x16x64x8): 适用于小型模型推理
   - Batch size: 4, Sequence length: 16, Hidden size: 64, Heads: 8
   - 迭代次数: 50次

3. **大型配置** (8x32x128x8): 适用于中等规模模型训练
   - Batch size: 8, Sequence length: 32, Hidden size: 128, Heads: 8
   - 迭代次数: 20次

4. **因果注意力配置** (4x16x64x8 causal): 测试因果掩码性能影响
   - 参数同中型配置，启用因果掩码
   - 迭代次数: 50次

##### 1.3 性能指标计算
- **前向传播时间**: 单次前向传播平均时间（毫秒）
- **反向传播时间**: 自动检测是否实现，未实现时标记为N/A
- **计算吞吐量**: 基于FLOPs估计的GFLOPs/s
- **参数数量**: 估算注意力层参数量（4 × hidden_size²）

##### 4.2 可选内存检测支持
在CMake中添加AddressSanitizer支持选项:
```cmake
option(BOAT_WITH_ASAN "Enable AddressSanitizer for memory error detection" OFF)
if(BOAT_WITH_ASAN)
    if(CMAKE_C_COMPILER_ID MATCHES "GNU|Clang")
        message(STATUS "Enabling AddressSanitizer")
        set(CMAKE_C_FLAGS_DEBUG "${CMAKE_C_FLAGS_DEBUG} -fsanitize=address -fno-omit-frame-pointer")
        set(CMAKE_EXE_LINKER_FLAGS_DEBUG "${CMAKE_EXE_LINKER_FLAGS_DEBUG} -fsanitize=address")
    else()
        message(WARNING "AddressSanitizer not supported for compiler ${CMAKE_C_COMPILER_ID}")
    endif()
endif()
```
