# 迭代1：完成MNIST autodiff示例重写

## 概述
本次迭代完成了MNIST自动微分示例的重写，创建了一个完整的基于Boat自动微分API的卷积神经网络训练示例。

## 完成的工作

### 1. 重写了 `mnist_autodiff.c`
- 使用Boat的自动微分API (`boat_var_conv`, `boat_var_pool`, `boat_var_dense`, `boat_var_relu`, `boat_var_softmax`等)
- 实现了完整的CNN架构：Conv2D → ReLU → MaxPool → Conv2D → ReLU → MaxPool → Flatten → Dense → ReLU → Dense
- 添加了交叉熵损失函数（使用`boat_var_log_softmax`和one-hot编码）
- 实现了训练循环、批量处理和准确率计算
- 集成了Adam优化器

### 2. 更新了构建配置
- 在 `examples/mnist/CMakeLists.txt` 中添加了 `mnist_autodiff` 目标
- 创建了 `examples/CMakeLists.txt` 以启用示例构建
- 重新配置CMake启用 `BOAT_WITH_EXAMPLES=ON`

### 3. 修复了框架编译问题
- 在 `src/layers/pool.c` 中添加了 `#define _USE_MATH_DEFINES` 和 `#include <math.h>` 以解决Windows上`INFINITY`未定义的问题
- 修复了 `mnist_autodiff.c` 中的API调用：
  - `boat_tensor_create()` 添加了 `BOAT_DEVICE_CPU` 参数
  - `boat_tensor_slice()` 添加了 `NULL` 步长参数
  - 添加了Windows平台的文件访问兼容性代码

- Boat库编译成功

```
无法解析的外部符号: boat_tensor_slice, boat_pool_layer_create, boat_pool_layer_free
```

### 问题分析
这些链接错误表明：
1. **函数导出不匹配** - 可能是`BOAT_API`/`BOAT_CALL`声明与定义不一致
2. **库构建不完整** - pool层函数可能未正确链接到boat库中

## 代码亮点

新的 `mnist_autodiff.c` 展示了Boat自动微分API的关键用法：
- **变量创建**: `boat_variable_create(tensor, requires_grad)`
- **前向传播**: `boat_var_conv()`, `boat_var_relu()`, `boat_var_pool()`, `boat_var_dense()`
- **损失计算**: 自定义交叉熵损失函数
- **反向传播**: `boat_variable_backward_full()`
- **优化器**: `boat_optimizer_step()`, `boat_optimizer_zero_grad()`
