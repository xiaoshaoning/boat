# 迭代3：解决MNIST数据获取问题并尝试训练

## 概述
本次迭代解决了迭代2中遗留的MNIST数据获取问题，通过实现多后端数据加载机制，确保了在各种网络环境和依赖库配置下都能获取MNIST数据。同时尝试运行完整的MNIST训练流程，发现了运行时段错误问题，为后续迭代提供了明确方向。

## 完成的工作

### 1. 实现多后端MNIST数据加载
- **问题分析**：原始`mnist`包因URL失效返回HTTP 404错误，且网络下载速度缓慢
- **解决方案**：在`mnist_data.py`中实现`load_mnist()`函数，支持多个数据源优先级：
  1. **TensorFlow/Keras**：首选，使用`tf.keras.datasets.mnist.load_data()`
  2. **Scikit-learn**：次选，使用`sklearn.datasets.fetch_openml('mnist_784')`
  3. **TorchVision**：第三选择，使用`torchvision.datasets.MNIST`
  4. **原始mnist包**：后备方案，保留原有接口
  5. **虚拟数据生成**：终极后备，当所有数据源都失败时生成随机数据用于测试
- **实现细节**：
  - 每个后端都封装在try-except块中，优雅降级
  - 保持统一的数据格式输出：(N, 28, 28)图像和(N,)标签
  - 添加环境变量`USE_DUMMY_DATA`强制使用虚拟数据，便于离线测试
- **影响**：彻底解决了MNIST数据获取对单一源的依赖，提高了示例的健壮性和可复现性

### 2. 数据准备与验证
- **数据生成**：使用`USE_DUMMY_DATA=1`成功生成小型测试数据集（100个训练样本，20个测试样本）
- **文件格式**：数据以二进制格式保存，包含形状信息和原始数据，符合C加载器期望
- **验证结果**：
  - 所有二进制文件成功生成：`train_images.bin`、`train_labels.bin`、`test_images.bin`、`test_labels.bin`
  - 小数据集版本也同时生成，便于快速测试
  - 数据形状正确：(N, 1, 28, 28)图像和(N,)标签

### 3. 构建验证与运行时问题发现
- **编译状态**：Boat框架和`mnist_autodiff`示例编译成功，无错误无警告
- **可执行文件**：`build/examples/mnist/Debug/mnist_autodiff.exe`存在且可加载
- **运行时问题**：执行训练时发生段错误（Segmentation fault）
  - 问题出现在程序启动后，可能在数据加载或第一次切片操作时
  - 错误代码139（段错误），表明内存访问违规
  - 初步分析可能与`boat_tensor_slice`实现或内存管理有关

### 4. 代码审查与问题定位
- **切片函数审查**：检查了`src/core/tensor.c`中的`boat_tensor_slice`实现
  - 函数逻辑基本正确，包含参数验证和错误检查
  - 支持连续张量的单位步长切片
  - 创建视图张量，正确管理引用计数
- **可能的问题点**：
  - 非连续张量切片支持有限（仅警告）
  - 非单位步长切片未实现（返回错误）
  - 内存对齐或数据类型大小计算可能存在问题
  - 父张量引用管理可能在某些边界条件下出错

### 段错误调试线索
1. **发生时机**：程序启动后立即崩溃，可能在：
   - 数据加载后的第一次张量操作
   - `boat_tensor_slice`调用时
   - 内存分配或释放时
2. **可能原因**：
   - 张量数据结构未正确初始化
   - 切片参数超出范围
   - 内存管理函数（`boat_malloc`/`boat_free`）问题
   - 数据类型大小计算错误

### 环境变量支持
- `USE_DUMMY_DATA=1`：强制使用虚拟数据，跳过所有网络下载
- `USE_FULL_DATA=1`：在C程序中强制使用完整数据集（如果可用）

## 经验教训

1. **数据源多样性**：依赖单一数据源（如原始mnist包）易受网络和URL变更影响
2. **优雅降级**：多层后备方案确保系统在各种环境下都能工作
3. **虚拟数据价值**：对于框架测试，虚拟数据足以验证流程，无需真实数据
4. **早期问题暴露**：小型测试数据集能快速暴露运行时问题，加速调试
5. **跨平台考虑**：Windows DLL加载需要正确设置PATH或复制DLL到可执行文件目录
