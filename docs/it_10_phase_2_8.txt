### Priority 1: Symbol Export Resolution (COMPLETE)

**Export Macro System**:
Created `include/boat/export.h` with platform-aware export/import macros:

```c
// Windows DLL export/import
#if BOAT_WINDOWS
    #ifdef BOAT_BUILDING_DLL
        #define BOAT_API __declspec(dllexport)
    #else
        #define BOAT_API __declspec(dllimport)
    #endif
#else
    // Non-Windows visibility attributes
    #if __GNUC__ >= 4
        #define BOAT_API __attribute__((visibility("default")))
    #else
        #define BOAT_API
    #endif
#endif
```

**Build System Updates**:
- Added `target_compile_definitions(boat PRIVATE BOAT_BUILDING_DLL)` to CMakeLists.txt
- Export macros automatically applied when building the shared library
- Maintains compatibility with `CMAKE_WINDOWS_EXPORT_ALL_SYMBOLS` fallback

**Header File Updates**:
- Updated `include/boat/format/pytorch.h` with `BOAT_API` on all public functions
- Updated `src/format/pytorch.cpp` function definitions with `BOAT_API`
- Added `#include "boat/export.h"` to relevant headers

**Expected Impact**:
- Resolves linking issues for test executables on Windows
- Enables proper DLL import/export semantics for all platforms
- Provides foundation for future cross-platform deployment

Known Issues and Limitations
----------------------------

### Current Limitations

1. **LSTM/GRU Implementation Depth**:
   - Forward/backward passes are placeholder implementations
   - Parameter extraction from PyTorch models not fully implemented
   - Bidirectional and multi-layer configurations need testing

2. **Transformer Support**:
   - Recognition of Transformer modules not yet implemented
   - Mapping between PyTorch Transformer and Boat attention layers pending
   - Complex architecture conversion requires detailed design

3. **Performance Optimizations**:
   - Zero-copy tensor sharing not yet implemented
   - Batch parameter conversion remains future work
   - Memory usage during conversion not optimized

4. **Testing Coverage**:
   - Limited testing of recurrent layer functionality
   - Need for comprehensive model zoo integration
   - Cross-platform validation pending

### Technical Constraints

1. **PyTorch API Complexity**:
   - LSTM/GRU parameters have complex layout (packed vs. unpacked)
   - Transformer modules have nested architecture requiring recursive conversion
   - Dynamic control flow models present additional challenges

2. **Memory Management**:
   - Shared tensor ownership between PyTorch and Boat requires careful design
   - Different memory allocators complicate zero-copy approaches
   - Lifetime management for intermediate conversion data

3. **Platform Compatibility**:
   - Export macro system needs validation on Linux/macOS
   - ABI stability across compiler versions requires attention
   - Cross-compilation scenarios need testing
