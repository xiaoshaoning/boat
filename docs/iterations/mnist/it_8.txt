# 迭代8：训练监控增强与数据标准化

## 概述
本次迭代基于迭代7的梯度修复和训练优化，进一步增强了训练监控能力并完善了数据预处理。主要实现了验证集评估、训练历史记录、数据标准化等功能，为模型性能分析和优化提供了更好的工具支持。

## 完成的工作

### 1. 验证集评估实现
- **训练/验证分割**：从60,000个训练样本中划分出5,000个样本作为验证集（前5,000个），剩余55,000个作为训练集。
- **验证评估函数**：新增`evaluate_model()`函数，用于在任意数据集上计算模型的损失和准确率。
- **训练循环集成**：每个训练周期（epoch）结束后自动在验证集上评估，显示验证损失和准确率。

### 2. 训练历史记录与可视化支持
- **指标跟踪数组**：动态分配数组存储每个epoch的训练损失、训练准确率、验证损失、验证准确率。
- **CSV文件导出**：训练结束后自动将训练历史保存为`training_history.csv`文件，包含以下列：
  - epoch: 周期编号（1-based）
  - train_loss: 训练损失
  - train_accuracy: 训练准确率
  - val_loss: 验证损失
  - val_accuracy: 验证准确率
- **内存管理**：正确释放训练历史数组和验证集张量，避免内存泄漏。

### 3. 数据标准化（均值减除与标准差归一化）
- **统计量计算**：基于训练集计算全局像素均值和标准差（单通道）。
- **标准化公式**：`x_standardized = (x - mean) / std`
- **一致变换**：训练集、验证集、测试集使用相同的训练集统计量，确保数据分布一致性。
- **计算结果**：MNIST训练集统计量为 mean=0.130700, std=0.308159（基于55,000个训练样本计算）。

### 4. 梯度精度初步分析
- **现状评估**：梯度检查测试在宽松容差（相对容差1e-2，绝对容差1e-3）下通过，表明梯度计算基本正确但存在数值误差。
- **误差来源分析**：可能的误差来源包括：
  1. 浮点运算累积误差（单精度float32）
  2. 有限差分法的截断误差与舍入误差平衡
  3. 卷积实现中的索引计算和累加顺序
- **改进方向**：
  - 使用双精度（float64）进行梯度检查基准
  - 优化卷积实现中的累加顺序（Kahan求和）
  - 分析梯度数值范围，识别异常值

## 技术细节

### 训练/验证集分割策略
- **简单分割**：使用前5,000个样本作为验证集，剩余55,000个作为训练集。
- **未来改进**：可添加随机洗牌和分层采样，确保类别分布均衡。
- **内存效率**：使用`boat_tensor_slice()`创建视图，避免数据复制。

### 数据标准化实现
1. **统计量计算**：
   ```c
   // 计算均值
   double sum = 0.0;
   for (size_t i = 0; i < total_pixels; i++) sum += data[i];
   float mean = (float)(sum / total_pixels);

   // 计算标准差
   double sum_sq = 0.0;
   for (size_t i = 0; i < total_pixels; i++) {
       float diff = data[i] - mean;
       sum_sq += diff * diff;
   }
   float std = (float)sqrt(sum_sq / total_pixels);
   ```

2. **标准化应用**：
   ```c
   for (size_t i = 0; i < total_pixels; i++) {
       data[i] = (data[i] - mean) / std;
   }
   ```

### 评估函数设计
- **批处理评估**：使用128的批次大小提高评估效率。
- **损失计算**：复用`cross_entropy_loss()`函数，确保与训练损失计算一致。
- **内存安全**：每个批次后及时释放临时变量。

### 训练历史管理
- **动态分配**：根据epoch数动态分配数组，适应不同训练配置。
- **文件格式**：CSV格式便于Python/MATLAB/Excel等工具分析。
- **错误处理**：文件创建失败时输出警告但不中断训练。
