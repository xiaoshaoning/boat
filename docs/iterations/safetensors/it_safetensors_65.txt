### 🔧 技术实施

#### 1. 段错误根本原因分析
- **问题1**: `boat_model_create()` 函数中未初始化计算图指针
  - 文件: `src/model/model.c:31-54`
  - 问题: `model->graph` 指针未初始化，直接检查 `if (!model->graph)` 导致随机行为
  - 修复: 创建新的计算图实例 `model->graph = boat_graph_create()`

- **问题2**: `boat_model_free()` 函数内存释放不完整
  - 文件: `src/model/model.c:83-115`
  - 问题: 多个空语句块，未实际释放层数据、节点数组、图、名称和用户数据
  - 修复: 完整实现内存释放逻辑，包括通过层操作释放层数据

- **问题3**: 测试中层包装器缺少操作函数表
  - 文件: `tests/test_model_graph_integration.c`
  - 问题: 手动分配的 `boat_layer_t` 结构体中 `ops` 字段为 `NULL`，导致 `boat_model_free()` 无法正确释放层数据
  - 修复: 在测试中添加层包装器操作函数表（仿照 `src/format/huggingface.c` 实现）

##### 2.2 模型释放函数修复
```c
// 修复前
void boat_model_free(boat_model_t* model) {
    if (!model) return;

    // 多个空语句块，无实际释放操作
    if (model->layers) {
        for (size_t i = 0; i < model->layer_count; i++) {
            boat_layer_t* layer = model->layers[i];
            if (layer) {
                if (layer->ops && layer->ops->free) {
                    // 空语句
                } else {
                    // 空语句
                }
            }
        }
    }
    // 其他空语句块...
}

// 修复后
void boat_model_free(boat_model_t* model) {
    if (!model) return;

    // 释放层数据
    if (model->layers) {
        for (size_t i = 0; i < model->layer_count; i++) {
            boat_layer_t* layer = model->layers[i];
            if (layer) {
                if (layer->ops && layer->ops->free) {
                    layer->ops->free(layer);  // 通过操作函数释放
                } else {
                    boat_free(layer);  // 后备释放
                }
            }
        }
        boat_free(model->layers);
    }

    // 释放节点数组（节点本身由图拥有）
    if (model->nodes) boat_free(model->nodes);

    // 释放计算图
    if (model->graph) boat_graph_free(model->graph);

    // 释放模型名称和用户数据
    if (model->name) boat_free(model->name);
    if (model->user_data && model->free_user_data) {
        model->free_user_data(model->user_data);
    }

    boat_free(model);
}
```
