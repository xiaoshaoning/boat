### 🔧 技术实施与修改

#### 1. 汇编级调试与根本原因发现
**工具**: 使用 `objdump -d` 反汇编 boat.dll 的 `.text` 节
**关键发现**:
- `boat_attention_layer_backward` 导出地址 `0x18000199C` 包含跳转指令 `jmp 0x18002afd0`
- 目标地址 `0x18002afd0` 的代码为延迟加载存根，执行 `xor %eax,%eax` 后返回 NULL
- 其他函数（如 `boat_attention_layer_forward`）跳转目标为实际函数体
- **根本原因**: 编译器优化将简单函数体消除，跳转存根指向通用返回 NULL 的存根

#### 2. 编译器优化控制
**修改文件**: `src/layers/attention.c`
- 添加 `__declspec(noinline)` 防止函数内联优化
- 实验性使用 `#pragma optimize("", off)` 和 `#pragma optimize("", on)` 包围函数定义
- 最终确定仅需 `__declspec(noinline)` 即可解决问题

#### 3. 函数体正确实现
**修改文件**: `src/layers/attention.c`
- 恢复正确的 `boat_attention_layer_backward` 实现
- 调用底层 `boat_attention_backward` 计算梯度
- 返回 query 梯度，释放未使用的 key/value 梯度
- 保持层接口一致性（单张量返回值）

**最终实现**:
```c
#ifdef _MSC_VER
__declspec(noinline)
#endif
BOAT_API boat_tensor_t* BOAT_CALL boat_attention_layer_backward(boat_attention_layer_t* layer,
                                                       const boat_tensor_t* grad_output) {
    boat_tensor_t* grad_query = NULL;
    boat_tensor_t* grad_key = NULL;
    boat_tensor_t* grad_value = NULL;

    if (boat_attention_backward((boat_attention_t*)layer, grad_output,
                                &grad_query, &grad_key, &grad_value)) {
        // Free unused gradients (key and value) as layer interface only returns query gradient
        if (grad_key) boat_tensor_free(grad_key);
        if (grad_value) boat_tensor_free(grad_value);
        return grad_query;
    }
    return NULL;
}
```

### 🔍 问题根本原因总结

经过深度调试，确认 Windows x64 DLL 边界问题的根本原因为：

1. **选择性优化**: 仅 `boat_attention_layer_backward` 函数受影响，因为其实现简单（仅返回输入指针）
2. **编译器优化消除**: MSVC 编译器将简单函数体优化消除，仅保留返回 NULL 的存根
3. **跳转存根错误**: DLL 导出跳转表指向编译器生成的通用存根而非实际函数体
4. **平台特异性**: Windows x64 调用约定统一，但编译器优化行为与 Linux/macOS 不同
5. **诊断特征**: 函数地址有效但函数体未执行，所有诊断代码（包括 `abort()`）未触发
