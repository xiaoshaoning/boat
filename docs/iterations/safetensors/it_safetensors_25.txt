### 🔧 完成的工作

#### 1. 张量引用计数系统修复
- **问题诊断**: 测试在 `boat_tensor_free(output)` 调用处阻塞，根本原因为重塑视图与原始张量之间的内存管理问题
- **根本原因**: `boat_tensor_reshape` 创建视图时共享数据指针但未增加原始张量引用计数，导致原始张量过早释放而视图仍引用已释放内存
- **解决方案**:
  - 扩展 `boat_tensor_t` 结构体，添加 `parent` 指针和 `is_view` 标志
  - 修改 `boat_tensor_reshape` 函数，将新张量标记为视图并增加原始张量引用计数
  - 重构 `boat_tensor_free` 函数，正确处理视图张量（不释放共享数据）并递归减少父张量引用计数
- **技术实施**:
  - `src/core/tensor.c:13-25`: 结构体添加 `parent` 和 `is_view` 字段
  - `src/core/tensor.c:123-127`: 初始化新字段为 `NULL` 和 `false`
  - `src/core/tensor.c:288-296`: 重塑函数设置视图关系并增加引用计数

#### 1. 张量内存管理改进
```c
// 修改后的 boat_tensor_t 结构体
struct boat_tensor_t {
    // ... 原有字段 ...
    boat_tensor_t* parent;    // 父张量（如果是视图）
    bool is_view;             // 是否是视图（共享数据）
};

// 重塑函数关键修改
new_tensor->is_view = true;
new_tensor->parent = (boat_tensor_t*)tensor;
boat_tensor_ref(new_tensor->parent);  // 增加父张量引用计数

// 释放函数关键修改
if (tensor->parent) {
    boat_tensor_unref(tensor->parent);  // 减少父张量引用计数
}
if (!tensor->is_view && tensor->data) {
    free_memory(tensor->data, tensor->device);  // 仅当拥有数据时才释放
}
```
