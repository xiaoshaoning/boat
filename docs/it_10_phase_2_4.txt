Technical Implementation Details
-------------------------------

### 1. Conv2d Layer Implementation

**Layer Structure**:
```c
struct boat_conv_layer_t {
    size_t in_channels;
    size_t out_channels;
    size_t kernel_size;
    size_t stride;
    size_t padding;
    bool use_bias;
    boat_tensor_t* weight;
    boat_tensor_t* bias;
};
```

**Conversion Logic**:
```cpp
// PyTorch to Boat Conv2d conversion
if (module_type.find("Conv2d") != std::string::npos) {
    // Extract weight tensor from parameters
    boat_tensor_t* weight_tensor = weight_it->second;
    const int64_t* weight_shape = boat_tensor_shape(weight_tensor);
    size_t out_channels = weight_shape[0];
    size_t in_channels = weight_shape[1];
    size_t kernel_size = weight_shape[2];

    // Create layer with extracted parameters
    boat_conv_layer_t* conv_layer = boat_conv_layer_create(
        in_channels, out_channels, kernel_size, stride, padding);

    // Set parameters with reference counting
    boat_conv_layer_set_weight(conv_layer, weight_tensor);
    if (bias_exists) boat_conv_layer_set_bias(conv_layer, bias_tensor);
}
```

**Key Features**:
- Automatic shape validation during parameter transfer
- Reference counting for efficient memory management
- Default stride=1, padding=0 (extensible to actual PyTorch values)
- Support for square kernels (height == width)

### 2. BatchNorm2d Layer Implementation

**Layer Structure**:
```c
struct boat_batchnorm2d_layer_t {
    size_t num_features;
    float eps;
    float momentum;
    bool affine;
    bool training;

    boat_tensor_t* weight;   // gamma
    boat_tensor_t* bias;     // beta
    boat_tensor_t* running_mean;
    boat_tensor_t* running_var;
};
```

**Conversion Logic**:
```cpp
// PyTorch to Boat BatchNorm2d conversion
if (module_type.find("BatchNorm2d") != std::string::npos) {
    // Extract parameters and buffers
    boat_tensor_t* weight_tensor = weight_it->second;  // if affine
    boat_tensor_t* running_mean_tensor = running_mean_it->second;
    boat_tensor_t* running_var_tensor = running_var_it->second;

    // Create layer with PyTorch defaults (eps=1e-5, momentum=0.1)
    boat_batchnorm2d_layer_t* bn_layer = boat_batchnorm2d_layer_create(
        num_features, 1e-5f, 0.1f, affine);

    // Set all parameters
    if (affine) {
        boat_batchnorm2d_layer_set_weight(bn_layer, weight_tensor);
        boat_batchnorm2d_layer_set_bias(bn_layer, bias_tensor);
    }
    boat_batchnorm2d_layer_set_running_mean(bn_layer, running_mean_tensor);
    boat_batchnorm2d_layer_set_running_var(bn_layer, running_var_tensor);
}
```

**Key Features**:
- Complete running statistics transfer from PyTorch buffers
- Support for both affine and non-affine BatchNorm variants
- Training vs inference mode handling
- Default PyTorch parameter values (eps=1e-5, momentum=0.1)

Challenges and Solutions
------------------------

### Challenge 1: PyTorch Parameter Extraction
- **Problem**: Conv2d and BatchNorm2d require extracting parameters from both `.parameters()` and `.buffers()`
- **Solution**: Separate handling for learnable parameters (weights/biases) and running statistics (mean/var)
- **Result**: Complete parameter transfer for all layer types

### Challenge 2: Layer Interface Consistency
- **Problem**: Different layer types require different parameter access patterns
- **Solution**: Standardized parameter setter functions for all layer types
- **Result**: Uniform API for model loading across Linear, Conv2d, BatchNorm2d

### Challenge 3: Performance vs Completeness Trade-off
- **Problem**: Full convolution implementation requires significant development
- **Solution**: Implemented basic forward pass with correct shape calculation for testing
- **Result**: Functional conversion pipeline with placeholder operations
