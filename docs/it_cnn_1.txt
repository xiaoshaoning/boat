Core Issue: Symbol Export Configuration (Resolved)
--------------------------------------------------
### Problem Description
The Boat library (`boat.dll`) compiled in `build_test_pytorch3/Release/` did not properly export C function symbols, causing linking failures for test programs.

### Root Cause
The export macro system (`include/boat/export.h`) was implemented in Phase 2.8, but the existing `boat.dll` was compiled before these changes. Additionally, header files `tensor.h` and `model.h` were not using the `BOAT_API` macro for function declarations.

### Solution Applied
1. Added `#include "export.h"` to `tensor.h` and `model.h`
2. Applied `BOAT_API` macro to all public function declarations in both headers
3. The library now needs to be rebuilt with `BOAT_BUILDING_DLL` defined to export symbols correctly

Build Instructions
------------------
### Option 1: Using Visual Studio (Recommended)
1. Open `build_test_pytorch3/boat.slnx` in Visual Studio
2. Add `BOAT_BUILDING_DLL` to project preprocessor definitions:
   - Right-click boat project → Properties → C/C++ → Preprocessor → Preprocessor Definitions
   - Add `BOAT_BUILDING_DLL` to the list
3. Rebuild boat.dll in Release configuration
4. Copy the new boat.dll to `external/mnist-cnn-digit-classifier/`

### Option 2: Using CMake Command Line
```cmd
cd D:\codes\boat
mkdir build_fixed
cd build_fixed
cmake -DBOAT_WITH_PYTORCH=ON -DBOAT_PYTORCH_PATH="D:/codes/boat/external/libtorch-python-cpu" -DBOAT_WITH_TESTS=ON ..
cmake --build . --config Release
```

### Option 3: Using MinGW (Alternative)
```bash
cd D:\codes\boat
make clean
make BOAT_BUILDING_DLL=1
# Note: May require adjustments to Makefile for Windows DLL compilation
```
