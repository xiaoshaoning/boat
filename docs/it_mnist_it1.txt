#### 潜在崩溃原因分析

| 可能原因 | 证据 | 概率 |
|----------|------|------|
| **Autodiff 函数未实现** | `boat_var_sum` 等函数返回 NULL | 高 |
| **张量操作错误** | `boat_tensor_slice` 可能返回无效指针 | 中 |
| **内存分配失败** | 大量张量创建可能导致内存不足 | 中 |
| **数据类型不匹配** | 预期 FLOAT32 但收到其他类型 | 低 |

### 🚧 根本问题分析

#### 问题1: Autodiff 核心函数未实现
- **位置**: `src/autodiff.c:543-546`
- **代码**:
  ```c
  boat_variable_t* boat_var_sum(boat_variable_t* a, int64_t* dims, size_t n_dims, bool keepdim) {
      (void)a; (void)dims; (void)n_dims; (void)keepdim;
      return NULL;  // 存根实现
  }
  ```
- **影响**: `cross_entropy_loss()` 函数依赖此函数计算损失，导致返回 NULL，后续操作崩溃。

#### 问题2: 错误处理不完善
- **位置**: `mnist_autodiff.c:cross_entropy_loss()`
- **现状**: 检查 `boat_var_sum()` 返回值但后续未正确处理 NULL
- **风险**: NULL 传播导致程序崩溃

#### 问题3: 构建配置问题
- **症状**: 重新编译时出现 `"无法打开包括文件: 'boat.h'"` 错误
- **原因**: CMake include 路径配置不正确或 boat 库未正确构建

#### 问题4: 层间数据流验证缺失
- **风险**: 卷积、池化、全连接层间的形状转换可能导致错误
- **示例**: Conv2 输出形状可能不正确，导致 Flatten 操作失败

### 💡 建议的解决方案

#### 短期修复（1-2天）

##### 1. 修复 Autodiff 核心函数实现
```c
// src/autodiff.c 中需要实现的函数
boat_variable_t* boat_var_sum(boat_variable_t* a, int64_t* dims, size_t n_dims, bool keepdim) {
    if (!a) return NULL;

    // 实现张量求和操作
    boat_tensor_t* input = boat_variable_data(a);
    boat_tensor_t* output = boat_tensor_sum(input, dims, n_dims, keepdim);

    if (!output) return NULL;

    // 创建输出变量，继承梯度要求
    return boat_variable_create(output, a->requires_grad);
}
```

##### 2. 在 mnist_autodiff.c 中添加防御性检查

if (!sum) {
