These components provide the foundation for automatic differentiation and computational graph operations.

Components Implemented:

1. Memory Management System (src/core/memory.c, include/boat/memory.h)
===================================================
Features:
- Reference-counted memory allocation with statistics tracking
- Device-aware memory management (CPU/CUDA interfaces)
- Aligned memory allocation for performance optimization
- Memory operation utilities (copy, set)
- Debug support with file/line tracking (optional)

Key Functions:
- boat_memory_allocate/boat_memory_free: Basic allocation with statistics
- boat_memory_allocate_zero: Zero-initialized memory allocation
- boat_memory_reallocate: Memory resizing with statistics update
- boat_memory_allocate_aligned: Alignment-aware allocation
- boat_memory_copy/boat_memory_set: Cross-device memory operations
- boat_memory_get_stats: Memory usage statistics retrieval

Statistics Tracking:
- Allocated bytes and blocks
- Peak memory usage
- Freed memory tracking
- Average block size calculation

Integration:
- Updated tensor.c to use new memory management functions
- Added forward declarations for proper compilation
- Fixed header dependencies (added stdio.h for FILE type)

2. Arithmetic Operations System (src/ops/arithmetic.c, include/boat/ops.h)
===================================================
Features:
- Element-wise operations with broadcasting support
- In-place operations for memory efficiency
- Scalar operations with type promotion
- Support for multiple data types (float32, float64, int32, int64)

Operations Implemented:
- Basic: add, sub, mul, div, mod
- In-place: add_, sub_, mul_, div_
- Scalar: add_scalar, sub_scalar, mul_scalar, div_scalar, pow_scalar
- Broadcasting: can_broadcast, broadcast_to (basic implementation)

Data Type Support:
- Floating point: float32, float64 (full support)
- Integer: int32, int64 (basic operations)
- Future support planned for low-bit types (bits2, bits1)

Broadcasting Implementation:
- Shape validation for broadcast compatibility
- Automatic output shape calculation
- Support for up to BOAT_MAX_DIMS dimensions
- Basic shape checking (full implementation pending)

Code Organization:
- Generic element-wise operation macros for code reuse
- Type-specific implementations for each operation
- Error handling for incompatible shapes/types

3. Graph Node Infrastructure (src/graph/node.c)

- Computational graph node abstraction
- Multiple node types (Variable, Operation, Constant, Placeholder, Output)

Node Types:
- VARIABLE: Trainable parameters
- OPERATION: Mathematical operations
- CONSTANT: Fixed values
- PLACEHOLDER: Input placeholders
- OUTPUT: Output nodes

Graph Structure:
- Dynamic node array management

- Basic graph creation and destruction
- Node reference counting integration

Edge Implementation:
- Directed edges with source/target nodes
- Edge direction tracking (forward/backward data flow)
- Reference counting for connected nodes
- Basic edge creation/destruction

Utility Functions:
