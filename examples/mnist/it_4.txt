# 迭代4：调试段错误并修复切片实现

## 概述
本次迭代针对迭代3中发现的段错误问题进行深入调试，通过添加详细日志定位崩溃点，修复了张量切片函数中的潜在问题，并发现了框架层实现中的梯度张量初始化缺失问题。虽然训练流程仍未完全通过，但为后续修复奠定了坚实基础。

## 完成的工作

### 1. 段错误定位与调试
- **添加系统化调试日志**：在`main()`函数、数据加载、模型创建、训练循环等关键位置添加`fprintf(stderr, ...)`打印
- **崩溃点定位**：发现程序在以下位置崩溃：
  1. 原始代码中`printf("Training samples: %zu\n", train_samples)`导致段错误（改为`fprintf(stderr, ...)`后解决）
  2. 模型创建成功，但在进入训练循环前崩溃，未到达切片操作
  3. 发现`boat_conv_layer_get_grad_weight()`返回`NULL`，表明卷积层的梯度权重未正确初始化

### 2. 张量切片函数修复
- **格式字符串修复**：将`boat_tensor_slice()`中的`%ld`改为`%lld`，兼容Windows平台`int64_t`类型
- **非连续张量处理**：将警告改为错误返回，避免对非连续张量进行不正确的切片操作
  ```c
  if (!tensor->is_contiguous) {
      fprintf(stderr, "Error: Slicing non-contiguous tensors not supported\n");
      boat_free(new_shape);
      boat_free(effective_step);
      return NULL;
  }
  ```
- **调试信息增强**：添加详细的参数打印，便于后续问题排查

### 3. 输出函数安全性改进
- **标准输出风险**：发现Windows环境下`printf()`可能导致段错误（可能因缓冲区或编码问题）
- **统一使用stderr**：将所有调试和进度输出改为`fprintf(stderr, ...)`，确保输出可靠性

### 4. 层初始化问题发现
- **梯度张量缺失**：`boat_conv_layer_get_grad_weight()`返回`NULL`，表明卷积层的梯度权重张量未创建
- **潜在影响**：优化器无法注册该参数，可能导致训练时梯度更新失败
- **根本原因**：框架层实现中可能未正确初始化梯度张量，需要检查`boat_conv_layer_create()`实现

### 关键发现
1. **段错误时序变化**：添加调试代码后，崩溃点从切片操作前移至模型创建后，表明内存布局变化影响崩溃位置
2. **梯度张量问题**：`conv1_grad_weight`为`NULL`，而`conv1_weight`有效，表明前向参数已初始化但反向参数缺失
3. **输出函数敏感性**：`printf()`与`fprintf(stderr, ...)`行为差异暴露平台特定问题

1. **格式兼容性**：消除Windows编译警告，确保跨平台一致性
2. **错误处理改进**：非连续张量切片明确返回错误，避免隐蔽的内存错误
3. **调试能力提升**：详细日志为后续问题排查提供基础设施

## 经验教训

1. **调试策略**：系统化添加日志比随机猜测更有效，尤其是对于间歇性段错误
2. **平台差异**：Windows环境对标准输出更敏感，生产代码应优先使用`fprintf(stderr, ...)`
3. **初始化完整性**：层实现必须同时初始化前向参数和梯度张量，缺一不可
4. **内存布局影响**：添加调试代码可能改变内存布局，从而改变崩溃位置，这既是优势也是挑战
