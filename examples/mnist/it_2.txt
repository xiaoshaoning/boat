# 迭代2：修复框架链接错误并实现tensor切片

## 概述
本次迭代解决了迭代1中报告的链接错误，修复了框架中的函数导出声明，并实现了缺失的 `boat_tensor_slice` 函数，使MNIST自动微分示例能够成功编译和链接。

## 完成的工作

### 1. 修复框架函数导出声明
- **问题分析**：链接错误 `无法解析的外部符号: boat_tensor_slice, boat_pool_layer_create, boat_pool_layer_free` 表明头文件中的函数声明缺少平台特定的导出宏。
- **解决方案**：在 `include/boat/layers.h` 中为所有层函数添加 `BOAT_API` 和 `BOAT_CALL` 宏：
  - 池化层函数 (`boat_pool_layer_*`)
  - 批归一化层函数 (`boat_batchnorm2d_layer_*`)
  - 激活层函数 (ReLU, Softmax)
  - 循环层函数 (LSTM, GRU)
  - 展平层函数 (`boat_flatten_layer_*`)
- **影响**：确保在Windows平台上正确导出DLL符号，解决链接器无法找到函数定义的问题。

### 2. 实现缺失的 `boat_tensor_slice` 函数
- **问题分析**：`boat_tensor_slice` 在头文件中声明但未在源代码中实现，导致链接错误。
- **解决方案**：在 `src/core/tensor.c` 中实现完整的切片功能：
  - 支持多维切片，包括起始索引、结束索引和可选步长
  - 正确处理连续张量的字节偏移计算
  - 创建视图（view）张量，共享父张量数据
  - 验证切片参数的有效性（索引范围、步长非零等）
- **实现细节**：
  - 计算新形状：`new_shape[i] = (end[i] - start[i]) / step[i]`
  - 计算行主序偏移量：`offset = sum(start[i] * stride[i]) * dtype_size`
  - 创建视图张量，设置 `is_view = true`，增加父张量引用计数
- **限制**：当前实现假设步长为1且张量连续，非连续张量和非单位步长支持为未来优化预留。

## 技术细节

### 函数导出修复
```c
// 修复前
boat_pool_layer_t* boat_pool_layer_create(size_t pool_size, size_t stride, size_t padding);

// 修复后
BOAT_API boat_pool_layer_t* BOAT_CALL boat_pool_layer_create(size_t pool_size, size_t stride, size_t padding);
```

### 切片实现关键逻辑
```c
// 偏移量计算（行主序）
size_t stride = 1;
size_t offset = 0;
for (int i = (int)ndim - 1; i >= 0; i--) {
    offset += start[i] * stride;
    stride *= tensor->shape[i];
}
offset *= dtype_size(tensor->dtype);
```

## 经验教训

1. **跨平台兼容性**：Windows DLL导出需要显式的 `__declspec(dllexport)` 声明，通过 `BOAT_API` 宏封装
2. **头文件一致性**：公共API函数必须在声明和定义中使用相同的调用约定和导出属性
3. **渐进实现**：复杂函数（如切片）可以先实现核心用例（连续张量、单位步长），再扩展功能
4. **构建验证**：每次API变更后应验证所有示例和测试的编译链接状态
