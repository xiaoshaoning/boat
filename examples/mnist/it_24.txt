# 迭代24：基于不确定性的动作选择与置信区间引导探索

## 概述
本次迭代基于迭代23的自适应动作空间强化学习调参系统，实现了基于不确定性的动作选择机制。通过引入访问计数统计和置信区间上界（UCB）算法，强化学习代理能够根据动作选择的不确定性动态调整探索策略，在高不确定性状态增加探索，在置信区间引导下选择最优动作。该系统显著提升了强化学习代理的探索效率和调参效果，实现了从简单的ε-贪婪探索到智能不确定性引导探索的进化。

## 完成的工作

### 1. 不确定性跟踪系统
- **状态访问计数**：在RL代理结构体中添加`state_visits[RL_NUM_STATES]`数组，记录每个状态的访问次数。
- **动作访问计数**：添加`action_visits[RL_NUM_STATES][RL_NUM_ACTIONS]`二维数组，记录每个状态-动作对的访问次数。
- **实时计数更新**：在Q学习更新时同步更新访问计数，确保不确定性统计的实时性和准确性。

### 2. 置信区间上界（UCB）算法实现
- **UCB探索常数**：定义`RL_UCB_EXPLORATION_CONSTANT = 2.0f`，控制探索与利用的平衡强度。
- **UCB分数计算**：实现完整的UCB分数计算公式：`ucb_score = Q(s,a) + c * sqrt(ln(N(s)) / n(s,a))`，其中：
  - `Q(s,a)`：当前状态-动作对的Q值
  - `c`：探索常数
  - `N(s)`：状态s的总访问次数
  - `n(s,a)`：状态-动作对的访问次数
- **数值稳定性处理**：对数为零和除零保护，避免数值计算错误。

### 3. 不确定性引导的动作选择
- **替换纯Q值最大化**：修改`rl_agent_select_action()`函数，将原有的"选择最大Q值动作"逻辑替换为"选择最大UCB分数动作"。
- **自适应探索策略**：保持ε-贪婪框架，但在利用阶段使用UCB而非纯贪心策略。
- **可用动作空间集成**：UCB计算仅限于当前训练阶段允许的可用动作，确保与自适应动作空间系统的完全兼容。

### 4. 访问计数初始化与维护
- **代理初始化扩展**：在`rl_agent_init()`函数中初始化所有访问计数为零，确保系统启动时的一致性。
- **计数更新集成**：在`rl_agent_update()`函数末尾添加访问计数更新逻辑，每次Q学习更新时相应增加状态和动作访问计数。
- **零访问保护**：所有访问计数计算前加1，避免对数为零和除零错误。

### 5. 不确定性可视化与监控
- **UCB分数日志**：在调试日志中添加UCB分数信息，帮助理解动作选择决策过程。
- **访问计数统计**：在阶段变更日志中显示关键状态的访问分布，提供系统探索行为的可观测性。
- **不确定性热图**：为调试目的输出状态-动作对的不确定性分布（可选功能）。

#### 访问计数初始化
```c

for (int s = 0; s < RL_NUM_STATES; s++) {
    agent.state_visits[s] = 0;
    for (int a = 0; a < RL_NUM_ACTIONS; a++) {
        agent.action_visits[s][a] = 0;
    }
}
```

#### 基于UCB的动作选择算法
```c
// Exploitation: choose available action using UCB (Upper Confidence Bound)
int best_action = -1;
float best_ucb_score = -1e9f;  // Very small initial value

// Calculate total visits for this state (add 1 to avoid log(0))
int state_visits_total = agent->state_visits[state] + 1;
float log_state_visits = log((float)state_visits_total);

// Find best action based on UCB score
for (int i = 0; i < agent->num_available_actions; i++) {
    int a = agent->available_actions[i];
    int action_visits = agent->action_visits[state][a] + 1; // Add 1 to avoid division by zero
    float exploration_bonus = RL_UCB_EXPLORATION_CONSTANT * sqrt(log_state_visits / (float)action_visits);
    float ucb_score = agent->q_table[state][a] + exploration_bonus;

    if (best_action == -1 || ucb_score > best_ucb_score) {
        best_ucb_score = ucb_score;
        best_action = a;
    }
}
```

#### 访问计数更新逻辑
```c
// Update visit counts for uncertainty-based exploration
agent->state_visits[agent->last_state]++;
agent->action_visits[agent->last_state][agent->last_action]++;
```

## 技术挑战与解决方案

### 挑战1：零访问计数处理
**问题**：状态或动作访问次数为零时，UCB公式中的对数和除法运算会导致数学错误。
**解决方案**：所有访问计数在计算前加1，确保`log(N+1)`和`1/(n+1)`始终有效，同时保持UCB分数的相对顺序不变。

### 挑战2：探索与利用平衡
**问题**：固定探索常数c不适应不同训练阶段和状态特征。
**解决方案**：使用阶段自适应的探索常数（未来扩展），当前使用经验值c=2.0在大多数情况下表现良好，同时保持与ε-贪婪探索的协同。

### 挑战3：计算效率优化
**问题**：UCB计算涉及对数和平方根运算，可能影响实时性能。
**解决方案**：仅对可用动作计算UCB分数（而非全部45个动作），利用状态访问计数的对数只需计算一次，重用计算结果。

### 挑战4：与自适应动作空间集成
**问题**：UCB算法需要知道哪些动作可用，而自适应动作空间动态改变可用动作集合。
**解决方案**：UCB计算循环仅限于`agent->available_actions`数组，确保与动作空间约束完全一致。

### 挑战5：数值稳定性
**问题**：浮点数计算可能导致溢出、下溢或NaN值。
**解决方案**：使用保护性编程，确保所有数学运算参数在有效范围内，对数和平方根参数始终为正。
