# 迭代14：学习率监控增强与训练曲线实时化

## 概述
本次迭代基于迭代13的内存使用统计功能，专注于学习率监控增强和训练曲线实时化。主要完成了学习率调度器集成、实时训练历史记录和批处理优化分析，为训练过程的精细化监控和优化奠定了基础。

## 完成的工作

### 1. 学习率监控增强
- **学习率调度器集成**：集成了Boat框架的`boat_step_lr_scheduler_create()` API，实现动态学习率调整策略。
- **调度器生命周期管理**：在模型结构中添加调度器字段，完整管理调度器的创建、更新和释放。
- **实时学习率监控**：在每个epoch开始时获取当前学习率并输出，在学习率变化时提供可视化反馈。
- **学习率历史记录**：将学习率值记录到训练历史CSV文件中，支持后续分析和可视化。

### 2. 训练曲线实时化
- **实时历史记录**：改造训练历史保存机制，从训练结束后批量写入改为每epoch实时追加，支持训练过程中实时监控。
- **时间戳集成**：在CSV记录中添加时间戳列，便于分析训练时间分布和进度。
- **实时文件刷新**：使用`fflush()`确保数据立即写入文件，避免缓冲延迟。

### 3. 批处理优化分析
- **批处理时间统计**：添加批处理级别的时间统计，计算平均批处理时间和批处理吞吐量。
- **详细性能指标**：在epoch输出中添加批处理性能指标（batches/s和s/batch），提供更细致的性能分析视图。
- **吞吐量关联分析**：将样本级吞吐量与批处理级吞吐量结合，全面评估训练效率。

## 技术细节

### 学习率调度器配置
```c
// 创建StepLR调度器：每1个epoch学习率乘以0.1
model->scheduler = boat_step_lr_scheduler_create(learning_rate, 1, 0.1f);

// 每个epoch后更新调度器
boat_scheduler_step(model->scheduler);
boat_scheduler_update_optimizer(model->scheduler, model->optimizer);

// 获取当前学习率
float current_lr = boat_optimizer_get_learning_rate(model->optimizer);
```

### 实时训练历史实现
```c
// 训练开始前打开文件
FILE* history_file = fopen("training_history.csv", "w");
fprintf(history_file, "epoch,timestamp,train_loss,train_accuracy,val_loss,val_accuracy,learning_rate\n");

// 每个epoch后实时写入
fprintf(history_file, "%d,%lld,%.6f,%.6f,%.6f,%.6f,%.6f\n",
        epoch + 1,
        (long long)time(NULL),
        train_losses[epoch],
        train_accuracies[epoch],
        val_losses[epoch],
        val_accuracies[epoch],
        learning_rates[epoch]);
fflush(history_file);  // 立即刷新缓冲区
```

### 批处理性能统计计算
```c
double epoch_time = (double)(end_time - start_time) / CLOCKS_PER_SEC;
double samples_per_second = epoch_time > 1e-6 ? (double)epoch_total / epoch_time : 0.0;
double avg_batch_time = epoch_time > 1e-6 ? epoch_time / num_batches : 0.0;
double batches_per_second = epoch_time > 1e-6 ? num_batches / epoch_time : 0.0;
```
